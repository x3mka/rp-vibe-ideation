# p4-004 — Add unit tests for `inthub-api`

**Status:** `planned`

Add comprehensive unit tests for the `inthub-api` library. Tests use a small, controlled test dataset (not the full inventory data).

## Test Coverage

- `makeApi()` — returns an object conforming to `IntHubApi`
- **Lookup by ID** — each `get<Entity>(id)` returns correct entity or `undefined` for unknown ID
- **List with filters** — `getProviderAccounts({ orgId })` filters correctly, returns all when no filter
- **Grouped queries** — `getProvidersByCategory()` groups correctly
- **Credential helpers** — `getExpiredCredentials()` and `getExpiringCredentials(days)` use date logic correctly
- **Integration graph** — `getIntegrationGraph(orgId)` returns correct nodes (ProviderAccounts) and edges (Integrations) for an org
- **Empty database** — all functions handle an empty `Database` gracefully
- **Health check queries** — `getHealthCheckRuns(bindingId)` returns runs in order

## Definition of Done

- Tests in `packages/inthub-api/src/lib/*.spec.ts`
- `pnpm nx test inthub-api` passes with zero failures
