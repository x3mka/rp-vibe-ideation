# p4-005 — Add unit tests for `inthub-data-inventory` (referential integrity)

**Status:** `planned`

Add tests that validate the inventory dataset's internal consistency. These act as a safety net — if someone edits fake data and breaks a reference, tests catch it.

## Test Coverage

- **No duplicate IDs** — within each entity array, all `id` values are unique
- **Foreign key integrity:**
  - Every `ProviderAccount.providerId` → exists in `providers`
  - Every `ProviderAccount.orgId` → exists in `orgs`
  - Every `Credential.providerAccountId` → exists in `providerAccounts`
  - Every `Credential.credentialTypeId` → exists in `credentialTypes`
  - Every `Config.providerAccountId` → exists in `providerAccounts`
  - Every `Config.configSchemaId` → exists in `configSchemas`
  - Every `Integration.orgId` → exists in `orgs`
  - Every `Integration.integrationTypeId` → exists in `integrationTypes`
  - Every `Integration.sourceAccountId` → exists in `providerAccounts`
  - Every `Integration.targetAccountId` → exists in `providerAccounts`
  - Every `HealthCheckBinding.healthCheckId` → exists in `healthChecks`
  - Every `HealthCheckRun.bindingId` → exists in `healthCheckBindings`
  - Every `ScheduledTask.orgId` → exists in `orgs`
- **Non-empty** — dataset has at least 2 orgs, at least 3 providers, at least 1 integration

## Definition of Done

- Tests in `packages/inthub-data-inventory/src/lib/data.spec.ts`
- `pnpm nx test inthub-data-inventory` passes with zero failures
