# p4-002 — Create `inthub-api` lib

**Status:** `planned`

Scaffold `packages/inthub-api` as an Nx TypeScript library. Implement the `makeApi(db: Database)` factory that returns an `IntHubApi` object with query/filter/lookup functions.

## API Surface

```ts
makeApi(db: Database): IntHubApi
```

### Functions by Domain

**Providers:** `getProviders()`, `getProvider(id)`, `getProvidersByCategory()`
**Orgs:** `getOrgs()`, `getOrg(id)`
**ProviderAccounts:** `getProviderAccounts(filter?)`, `getProviderAccount(id)`
**Credentials:** `getCredentials(filter?)`, `getCredential(id)`, `getExpiredCredentials()`, `getExpiringCredentials(withinDays)`
**Configs:** `getConfigs(filter?)`, `getConfig(id)`
**IntegrationTypes:** `getIntegrationTypes()`, `getIntegrationType(id)`
**Integrations:** `getIntegrations(filter?)`, `getIntegration(id)`, `getIntegrationGraph(orgId)` — returns `{ nodes, edges }`
**HealthChecks:** `getHealthChecks(filter?)`, `getHealthCheckBindings(filter?)`, `getHealthCheckRuns(bindingId)`
**ScheduledTasks:** `getScheduledTasks(filter?)`
**Provisioning:** `getProvisionings(filter?)`

Each domain is implemented in its own file. `make-api.ts` composes them into the `IntHubApi` interface.

## Definition of Done

- `packages/inthub-api/` exists with Nx project structure
- `makeApi()` factory accepts any `Database` and returns `IntHubApi`
- All query functions handle missing data gracefully (return `undefined` or empty arrays)
- `pnpm nx build inthub-api` succeeds
- Depends on `@rp-vibe-ideation/inthub-entities`
