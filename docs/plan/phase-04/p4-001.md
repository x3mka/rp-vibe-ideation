# p4-001 — Create `inthub-entities` lib

**Status:** `done`

Scaffold `packages/inthub-entities` as an Nx TypeScript library. Define all IntHub entity interfaces, types, enums, and the `Database` interface.

## Entity Files

Each entity gets its own file under `src/lib/`:

| File | Exports |
|---|---|
| `provider.ts` | `Provider`, `ProviderCategory` (enum) |
| `org.ts` | `Org`, `OrgStatus` (enum) |
| `provider-account.ts` | `ProviderAccount`, `ProviderAccountStatus` (enum) |
| `credential-type.ts` | `CredentialType` |
| `credential.ts` | `Credential`, `CredentialStatus` (enum) |
| `config-schema.ts` | `ConfigSchema`, `ConfigPurpose` (enum) |
| `config.ts` | `Config`, `ConfigStatus` (enum) |
| `integration-runtime.ts` | `IntegrationRuntime`, `RuntimeType` (enum), `RuntimeStatus` (enum) |
| `integration-type.ts` | `IntegrationType`, `IntegrationTypeStatus` (enum) |
| `integration.ts` | `Integration`, `IntegrationStatus` (enum) |
| `provisioning.ts` | `Provisioning`, `ProvisioningStatus` (enum) |
| `health-check.ts` | `HealthCheck`, `TargetEntityType` (enum), `Severity` (enum) |
| `health-check-binding.ts` | `HealthCheckBinding` |
| `health-check-run.ts` | `HealthCheckRun`, `RunStatus` (enum) |
| `scheduled-task.ts` | `ScheduledTask`, `TaskType` (enum), `TaskStatus` (enum) |
| `database.ts` | `Database` — the top-level interface containing arrays of all entities |

Barrel export from `src/index.ts`.

## Definition of Done

- `packages/inthub-entities/` exists with Nx project structure
- All entity interfaces and enums are defined
- `Database` interface aggregates all entity arrays
- `pnpm nx typecheck inthub-entities` succeeds
- Package is importable as `@rp-vibe-ideation/inthub-entities`

> **Note:** This project uses source-first TypeScript packages (exports point to `./src/index.ts`). There is no separate build step; `typecheck` runs `tsc --build --emitDeclarationOnly` and validates the package.
